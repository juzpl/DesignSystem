# Publishing design.juz.pl

Ta dokumentacja opisuje przygotowanie publicznej wersji design systemu. Publikacja do GitHub lub na serwer jest osobnym krokiem i nie jest wykonywana automatycznie.

## Build

```bash
npm run build
npm run build-storybook
```

Wynik Storybooka trafia do:

```txt
storybook-static/
```

To ten katalog nalezy wystawic pod domena:

```txt
https://design.juz.pl
```

## Hosting

Minimalne wymagania hostingu:

- statyczny hosting plikow HTML/CSS/JS,
- HTTPS,
- przekierowanie `design.juz.pl` na katalog `storybook-static`,
- cache dla assetow z hashem,
- brak cache albo krotki cache dla `index.html`.

Przyklad konfiguracji nginx:

```nginx
server {
  server_name design.juz.pl;
  root /var/www/design-juz-pl/storybook-static;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location /assets/ {
    expires 30d;
    add_header Cache-Control "public, immutable";
  }
}
```
