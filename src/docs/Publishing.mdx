# Publishing DesignSystem

Ta dokumentacja opisuje przygotowanie publicznej wersji design systemu. Publikacja jest osobnym krokiem i nie jest wykonywana automatycznie.

## Build

```bash
npm run build
npm run build-storybook
```

Wynik Storybooka trafia do:

```txt
storybook-static/
```

## Hosting

Storybook to statyczne pliki — wystawiamy je na GitHub Pages z brancha `gh-pages` (workflow `.github/workflows/pages.yml` robi to automatycznie po pushu na `main`).

Minimalne wymagania alternatywnego hostingu:

- statyczny hosting plików HTML/CSS/JS,
- HTTPS,
- cache dla assetów z hashem,
- brak cache albo krótki cache dla `index.html`.
