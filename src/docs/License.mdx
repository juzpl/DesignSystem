## TL;DR

Możesz używać design.juz.pl w projektach **komercyjnych i niekomercyjnych**, modyfikować go i redystrybuować. W zamian wymagamy widocznej **atrybucji w stopce** wdrażanego produktu.

## Co możesz

- ✅ Używać komercyjnie i niekomercyjnie
- ✅ Modyfikować komponenty, tokeny, stories
- ✅ Redystrybuować, w tym zmodyfikowane wersje
- ✅ Łączyć z własnym kodem
- ✅ Wybierać tylko części Design Systemu

## Czego wymagamy

W stopce wdrażanej aplikacji wstaw widoczną, klikalną atrybucję:

```html
<footer>
  UI based on
  <a href="https://github.com/erp77flow/design.juz.pl"
     target="_blank" rel="noopener">design.juz.pl</a>
</footer>
```

Rozszerzona forma (rekomendowana w README projektu pochodnego):

```txt
UI based on design.juz.pl
Author: Rafał Mazur
Owner: Alfabet sp. z o.o. — https://www.alfabet.lu
Repo: https://github.com/erp77flow/design.juz.pl
```

Atrybucja musi być **widoczna** dla użytkownika końcowego, **klikalna** i **czytelna** (bez ukrywania przez `display:none`, `opacity:0` itp.).
